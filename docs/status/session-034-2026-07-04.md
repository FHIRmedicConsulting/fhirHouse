## Patient Access finding (EOB surface)

The claims resources (EOB/Coverage/Claim/ClaimResponse) were **already** R4-Core + patient-compartment
members, so search / `_include` / `$everything` / `$export` already reached them. The only real gap
was the two CARIN Blue Button EOB search params R4 Core omits — `type` and `service-date` — added via a
small additive `profile-search-params` layer (expressions resolve to primitives, so no engine change;
`service-date` is service-start-based, full Period-overlap deferred). CARIN/PDex profile *conformance*
remains IG-install (L5).

## Consent model (both env-gated, OFF by default; production = deploy gate)

- **Payer-to-Payer = OPT-IN** (`RONIN_P2P_CONSENT_REQUIRED`): after a unique `$member-match`, require
  an active *permit* `Consent` (category `payer-to-payer`) before disclosing the identifier → else 403.
- **Provider Access = OPT-OUT** (`RONIN_PROVIDER_ACCESS_OPTOUT`): `Group/$export` (attribution Group)
  drops patients with an active *deny* `Consent` (category `provider-access`).
- Category codes are operator-configurable (`RONIN_P2P_CONSENT_CATEGORY`,
  `RONIN_PROVIDER_ACCESS_CATEGORY`). Rich policy (DS4P/provisions/purpose-of-use) stays with the
  heritage consent engine (ADR-0030/0018).
